# docs/theme-foundations/

**Purpose:** the durable home for theme doctrine and per-theme foundations — everything a future agent needs to open a theme build without reconstructing decisions from chat history. Themes are laid over the app after the current bones work completes; nothing in this folder is armed.

**Reading order for a fresh agent:**
1. `docs/open-threads.md` — the living ledger. Always first; this folder is doctrine, the ledger is state.
2. `theme-arc.md` — the cross-theme progression, the orange thread, the access model. Every theme answers to it.
3. The theme's own foundations doc (currently: `volant-foundations.md`) — statuses inside are explicit: LOCKED (Nick's rulings), STANDING (committee completions, strike-able by Nick only), OPEN (hardware dials and pending decisions).
4. The theme's normative visual reference (currently: `volant-rc2.html`) — parity target for any build.
