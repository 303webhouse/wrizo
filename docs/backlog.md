## Mode-aware editor — Phases 2-3 + typewriter fade shipped
Brief: [mode-aware-editor-brief.md](mode-aware-editor-brief.md); built to the prototype `apps/desktop/scratch/wrizo-modes-hybrid.html` (feel source). New `components/ModeStage.tsx` wraps the editor in the writing-studio chrome; `useChromeDissolve.ts` is the push-back-to-writing engine; `store/writingSettings.ts` is the persisted gear.
- **Push-back-to-writing engine (`useChromeDissolve`).** Dissolve on keystroke (rails/format-pen bar → faint floor `--fade-min`, top bar → 0); **fixed timing 3-min wait → ~2-min slow fade-in** (the prototype's Preview/Real toggle was a demo — NOT shipped); explicit summon (viewport edge / `Esc` / tap-off-text) returns chrome fast (~0.4s) and cancels the slow timer; **casual pointer movement does NOT summon**. Drives `WritingSession` so the global header recedes in step; replaces `useChromeFade` in QuickSprint (the hook stays for other surfaces).
- **Glow + progress.** Ambient ember behind the page, opacity/scale tied to progress, **eased** (`pow(p,.55)`) so early words warm the room; blooms as the rails dissolve (they occlude it at rest). Progress bar under the page **stays visible while writing** (feedback, not chrome): metric Words / Time (first-write→now) / Off, soft goals (`WORD_GOAL=250`, `TIME_GOAL=25m`).
- **Rails + AI frame (frames only).** Left: Journal = capture (Spark deck / Fragments / Send→Drawer), Drafting = tools (Structure / Pages / Notes / Find) — labels + hoverable stubs. Right AI: **SEALED** shutter in Journal ("journal is yours alone"), **OPEN** empty assist surface elsewhere; **drawer-open animation** on Journal→other only.
- **Format / pen bar.** Drafting: working **Bold / Italic / Underline / bulleted-list / indent / outdent** (`document.execCommand` on the free-edit surface); heading / quote / ⋯more are stubs. Journal: **3 inks** (black-brown / red / blue) set the editor text + caret colour (new `penColor` prop on `ForwardOnlyEditor`); nib selector stub.
- **Settings (real, persisted behind the gear).** Progress (Words/Time/Off), Fade depth (Partial 8% / Full 0), Top bar (Dissolve / Dim = stays clickable), Typewriter (On/Off). `store/writingSettings.ts` (module store + `useWritingSettings` hook).
- **Typewriter fade (default on, toggle-able).** The lit page is a fixed-height scroll viewport; a top mask-gradient fades history, and a MutationObserver keeps the caret/active line at ~62% of the viewport so earlier lines ride up and dissolve as the writer moves down. Works in both modes (caret-rect band-scroll; falls back to last-line). Off → no mask, normal scroll.
- **Layout:** adapted into QuickSprint's flow (timer/finish/beat intact), 3-column centred stage; **rails hide < 900px** (phone/tablet-portrait get the page — the writing surface is what matters on small screens). Verified in-harness (18 checks: rails/sealed-AI/pen inks/glow/dissolve-on-write/Esc-resurface/progress/Drafting-open+bold/settings+persist/typewriter toggle/**IME no-regression**); selftest green.

## Cleanup: Bug 1 (mobile IME) + Bug 2 (New page) — shipped
- **Bug 1 (mobile text vanishing) — fixed.** `ForwardOnlyEditor` had no IME/composition handling: `beforeinput` preventDefault clobbered soft-keyboard/swipe/autocorrect composition, so mobile-typed text vanished. Fix: (1) hand off to the IME during composition — skip preventDefault on `insertCompositionText`/`isComposing`, commit the finalized text on `compositionend`; (2) render the editable via `innerHTML` (dangerouslySetInnerHTML) so each render fully replaces the DOM — the post-composition render wipes the browser's draft nodes (no duplication), and React skips the update when the html string is unchanged (an unrelated re-render mid-composition can't clobber the composing text); (3) don't re-anchor the caret mid-composition. Verified in-harness with **real CDP composition events** (`Input.imeSetComposition` + commit): composed text lands, mixed keyboard+IME appends, runway still strikes. *Real-device confirmation still wise, but the CDP path mirrors mobile IME.*
- **Bug 2 (New page opened previous writing) — fixed.** `/sprint` with no project reuses the shared `'scratch'` draft, so "New page" reseeded the last text. Desk's "New page" now `clearDraft('scratch')` before navigating → blank every time ("Keep writing" still resumes it). Proper per-document persistence (unique page ids, no data loss) is the deferred big-architecture work.
- Harness gained an `ime()` helper (CDP composition) for future mobile-input testing.
