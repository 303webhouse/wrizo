## BUG ‚Äî Journal ink capture (pen ‚Üí text) ‚Äî fixed
Brief: fix-journal-ink. Stylus drawing in the Journal converted to typed text with no ink retained.
- **Bisect (confirmed first, not blind):** the routing hypothesis was DISPROVEN ‚Äî Journal entries open at `/journal/:id` (the ink view), never the text-only PageEditor; B4's only journal change was the additive greyed-tab strip (above the sheet, no stacking impact); strokes still persist. A synthetic CDP pen already produced a stroke (not text), so routing + canvas stacking were fine.
- **Root cause:** the sheet's pen listeners ran in the **bubble** phase, but on an authored page the editable text node is the pen's event TARGET ‚Äî so OS stylus handwriting (which converts the pen to text) won at the target before the bubble-phase ink handler could `preventDefault`. (Synthetic pens don't trigger OS handwriting, which is why it "passed" headlessly.)
- **Fix:** move the pen interception to the **capture phase** (`{ capture: true }`) + `stopPropagation()` for pen in `onDown`, so the sheet grabs a pen FIRST and keeps it off the editable (no caret, no handwriting) ‚Äî the stroke goes to the ink canvas. Non-pen (finger/mouse) still falls through, so caret/typing are unaffected. No `strokes` storage touched.
- Verified in-harness (6 checks: ink canvas mounts, pen ‚Üí stroke not text, canvas painted, stroke persists across reload, existing ink entry still renders). Selftest green. **Real-stylus confirmation still wise** ‚Äî the headless harness can't simulate OS handwriting, so the capture-phase hardening is verified structurally + against synthetic pens, not against a physical stylus.
