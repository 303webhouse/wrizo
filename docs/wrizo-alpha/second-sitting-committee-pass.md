## CHAMBER 1 — the per-mode tool strips

**The frame (Experts).** The modes are stages of a practice, not
feature bins; each strip should hold ONLY what that stage's work
needs (the two-action law and anti-fiddle identity both point the
same way). The storage law under everything: `entry.text` stays
plain text with markdown conventions — the strips must never demand
a storage model the page can't honestly keep.

**The recommendation (Architects), mode by mode:**
- **Free Write:** the typewriter cluster ONLY — Nick's ruled naming:
  the Typewriter icon opens **Forward Momentum / Text Fade / Page
  Scroll** toggles; "Controls" and "Forward lock" as labels retire.
  Plus the goal block. Nothing else — this stage produces words.
- **Draft:** the markdown set, extended by exactly ONE convention:
  **strikethrough (`~~`)** joins bold/italic/headings/spacing — a
  plain-text-compatible amendment to the frozen set, requiring
  Nick's word by name since the freeze was his ruling. The em dash
  rides (already built). Dimmed-syntax register unchanged.
- **Revise:** typography lands here — with the chamber's one honest
  push-back, below. Font choice and size as PAGE-LEVEL presentation
  (the whole page's dress), alignment/indentation deferred until a
  real need names itself. **Custom font upload** wakes here from the
  parked user-authored-identity cluster: progressive disclosure,
  sought out inside Revise's type controls, never offered.
- **Workshop:** out of scope this pass (its identity belongs with
  the Tutor's growth and the social arc; nothing invented today).
- **Publish:** the one clipboard door (Copy My Words / Copy
  Formatted) and export — already law (copy-out Publish-only).

**The push-back (Opposition), named honestly:** Nick asked for
per-selection font and size changes. The chamber recommends AGAINST
span-level typography: `entry.text` cannot carry font/size spans
without inventing a rich-text format (the storage cost the pass
must name loudly — it is a schema-class decision wearing a toolbar
button's clothes), and span-level fiddling is the anti-fiddle
identity's named enemy (the iA lineage: one page, one voice).
PAGE-level typography delivers the felt want (make this page mine)
at zero storage cost. Bold/italic/strikethrough remain the
span-level set because markdown carries them honestly. **Nick
ratifies or overrules this specifically** — an overrule reopens it
as a schema-flagged design, not a strip tweak.

**Progress Bar (Nick's ruled rename of "Instruments"):** keepable
in the strip, disableable, or docked at the screen's foot under the
page — a three-way preference, per-writer, presentation-only.

**Lands as:** ticket **TS1** (strips + renames + strikethrough +
Progress Bar placements + page-level type controls + font upload).
Zero schema throughout.

## CHAMBER 2 — the staged vanish

**The frame.** Nick's desk metaphor is the law's shape: deep in
writing, you remain dimly aware of the desk's things without being
addressed by them — then, truly in the zone, the room is gone.
Instant-gone was wrong (his verdict); permanent-ghost is also wrong
(his restated want). The chamber's design is ONE engine with three
states, replacing every bespoke dissolve:

**PRESENT → GHOST → GONE.**
- First keystroke: chrome drops to a GHOST (barely perceivable,
  one shared opacity token) — the strip included, shutting like a
  drawer (a short slide + fade; reduced-motion honored).
- Sustained writing crosses a threshold — **line-based, not
  clock-based** (writing-native: fresh line-equivalents written
  since last chrome touch; working constant ~8, tuned by Nick's
  hand) — and GHOST fades to GONE.
- Any approach (the existing dwell/summon, now trusted-event-
  proven) returns chrome to PRESENT. Docked panels are EXEMPT by
  their own standing law; the first-run threshold is EXEMPT
  (purity); the paper never moves at any transition.

**The canon touch, drafted for ratification — A19, the Staged
Vanish Law:** availability is constant (approach always summons);
PRESENCE stages (present → ghost → gone) through one shared engine;
no surface may implement a private dissolve. This formally amends
the cascade strip's "always present" letter — its availability
stays absolute; its visual presence now stages with everything
else.

**Lands as:** ticket **V1** — recommended FIRST of this sitting's
tickets (it is Nick's oldest standing felt complaint). Zero schema.

## CHAMBER 3 — the cards (titles, tags, the footer)

**The frame.** Inheriting A16: cards are arrangement citizens; any
metadata must serve arranging and finding, never become a form to
fill. "Highly flexible, easy to figure out" translates to: reuse
the page's own patterns, add no ceremony.

**The recommendation:**
- **Title = the first line.** No separate title field, ever — a
  card's first line is its title, exactly as a page's is (the house
  convention, zero storage, zero ceremony). The popup needs no
  title box; the card face already renders it.
- **Tags: `box.tags` (optional string array on the Box)** — the
  boxes-jsonb precedent (zero schema), mirroring `entry.tags` so
  pages and cards share ONE tag vocabulary across the whole app.
  Edited in the card popup as a quiet tag row (the page's own tag
  UI pattern, reused). Rendered in the footer.
- **The footer (already built, toggleable) gains its contents:**
  tags + connections (present) + — when B3 arrives — the card's
  structure position (Act/Scene, beat name) via the plan-card link,
  a field B3 defines, not this pass. Nothing else invented; the
  footer is a caption, not a dashboard.
- **Finding: the board lens** — one quiet control on user boards:
  filter visible cards by tag (dim non-matching, never remove —
  arrangement is sacred). Recommended to ride the same ticket as
  tags so the field is useful the day it exists.

**The trims (Opposition):** no card colors, no priorities, no
statuses, no due-anything — every one is a gamification-adjacent
form field the identity forbids until a real writing need names it.

**Lands as:** ticket **C1** (tags + popup tag row + footer contents
+ the board lens) — can ride adjacent to B2 (same organizational
territory) or standalone; zero schema.
