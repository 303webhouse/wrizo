# Board Modes — second pass · the taxonomy and the flip · 2026-07-21

*Commissioned by Nick, superseding the first pass's SCATTER/SEQUENCE
recommendation (T1/T2 of that pass are overtaken; its trims and its
doors-not-modes law survive and are tested hard below). His proposal:
Storyboard, Outline, Research/Web, Open — each mode's menu offering
deck presets that render differently per mode — plus a Write tab on the
Board and a Plan tab on the Page, making each page "a kind of project"
with two faces, switchable and eventually side-by-side.*

## PASS ONE — proposals

**Dr. Varga — the load-bearing idea is the projection, and it must be
named before anything is built.** Nick's "Stasis Theory Argumentation"
example — one deck rendering as cards on a Storyboard and as an
editable sectioned document in Outline — is the whole architecture:
**decks are data; modes are projections.** A deck stops being "a set of
cards" and becomes a structured description (sections, beats, prompts,
order) that each mode renders in its own grammar. Get this right and
every future deck works in every future mode for free; get it wrong and
each mode needs its own deck library forever. This is the committee's
one non-negotiable: BM1 builds the projection seam even if v1 has only
two projections.

**The Rhetorician — the taxonomy is genre theory, and it's correct.**
Four modes are four discourse situations: narrative sequence
(Storyboard), hierarchical argument (Outline), associative web
(Research), and pre-genre openness (Open). That's not feature sprawl —
it's the honest observation that a novelist, an essayist, and a
worldbuilder arrange differently. One naming ruling: **"Project" stays
retired** (B2.1's word swap holds); the concept is "a page and its
plan," never a Project. And for Research's name, the bench proposes
**Commonplace** — the writer's own centuries-old word for the book
where sources, quotations, and gathered material live. Web as
runner-up. Nick's lexicon seam rules.

**The Writing-Center Director — the flip is NOT the thing I killed,
and the distinction must be drawn exactly.** Last pass I ruled exits
are doors, not modes, and I stand on it: a tab that navigates somewhere
*ambiguous* is a trap. But Nick's Write⇄Plan is a **paired flip** — a
Page's Plan face and a Board's Write face are 1:1, the destination is
never ambiguous, and the gesture is "turn the sheet over," not "go
somewhere." Ruling: the flip is lawful and good — **rendered at the
bar's end as a distinguished control, visually apart from the mode
tabs**, on both surfaces symmetrically. It is a hinge, not a fifth
mode; putting it among the tabs would teach writers that tabs sometimes
teleport, and then every tab is suspect. FX10's named return is this
control's ancestor and evolves into it on paired boards without waste.

**The Expressivist — Open is the default and the flip must never
nag.** A writer who never plans must never meet the planning face
uninvited: no empty board auto-created, no "set up your plan!" prompt,
no dot on the Plan control. **The plan face is born lazily, on first
flip, and not before.** Page is Primary means the paper is home and
planning is a door the writer opens — or never does.

**The Grammarian — Outline earns its keep only if it's real
structure.** Roman-numeral thinking is hierarchy: sections containing
points containing evidence. If Outline is just cards in a vertical
stack it's Storyboard rotated 90°. The projection must render genuine
nesting — and Nick's own instinct that Outline reads "more like a
preset Page with sections" is right, which raises the pass's sharpest
architectural question, handed to the Architects.

**The Novice — I want the flip to be instant and I want to see both.**
"When I'm drafting and forget where the scene goes, I want my board
back in one click with everything where I left it — and honestly,
sometimes I want the map open *beside* the page. But if switching is
fast enough I'd survive without the split for a while."

**The Architects — the shape, and three hard facts.**
1. **The pairing is schema.** A 1:1 page⇄board link is a relation the
   database must hold — a schema ticket under the standing rule, Nick's
   explicit go at merge, no exceptions. Lazy creation on first flip;
   existing boards gain Write faces by explicit pairing, never by
   guess; derived Journal/Shelf membership untouched.
2. **The app already has a Plan.** M1's StoryPlan/beats machinery and
   the cascade's Plan panel exist today. Nick's Plan face must
   *integrate or supersede* — two plan systems is a constitutional
   defect. BM1's brief requires a code-level reconciliation before a
   line is written; the likely answer is that StoryPlan becomes the
   Storyboard projection's data and the Plan panel becomes a door to
   the plan face, but that is a finding to verify, not assume.
3. **Outline-as-document blurs Board into Page — resolve it at the
   projection layer.** Outline is a *projection of the same deck data*
   into an editable sectioned document surface. Same data, page-like
   renderer. This keeps one deck library and avoids a third species of
   surface. Sequencing: **BM1 = the pairing + the flip + Open +
   Storyboard** (projection seam cut, two projections live); **BM2 =
   Outline** (the document projection); **BM3 = Commonplace** (v1
   connections derived from shared box.tags — the zero-schema field
   ratified at the Boards committee — explicit link-drawing later);
   **BM4 = side-by-side** (a genuine layout-engine ticket: two live
   surfaces, the Tutor's room, FX2's clearance law all renegotiated —
   real design, not a checkbox). The flip ships first precisely so the
   split can wait.

**Marketing — opposition duty flagged for pass two:** growth will want
the Plan control badged ("2 beats uncovered"), the flip to auto-create
boards ("activation!"), and Commonplace to bless web clipping. All
three go to the block.

## PASS TWO — opposition and trims

**Scope skeptic: four modes, a pairing schema, a projection engine, and
a split view — this is a quarter's work proposed in a sentence.**
Sustained as phasing, not as veto: the BM1–BM4 ladder above is the
answer, each rung shippable alone, the projection seam paid once in
BM1. Nothing lands before vacation; the decision lands now so E1, FX10,
and TU5 don't fight it.

**Canon skeptic: doesn't the flip violate Page is Primary?** No —
it completes it. The board becomes the page's *planning face*,
subordinate by construction: the page is home, the plan is a room off
it. What would violate the canon is the reverse — boards spawning
obligatory pages, or the plan face greeting new writers first. Both
named unlawful here.

**Growth's three, killed on the record:** badges on the flip control
(a knock — dead); auto-created plan faces (uninvited planning — dead);
web clipping into Commonplace (the anti-slop rail's paste-in target —
dead at the rail, forever, and Commonplace's own decks make the lawful
alternative).

**The Grammarian's floor, adopted:** Outline ships only when it can
nest. A flat list projection does not open the mode.

## RECOMMENDATION

Adopt the four-mode taxonomy — **OPEN (default) · STORYBOARD · OUTLINE
· COMMONPLACE** (names at Nick's seam) — with **decks as data, modes as
projections** as the constitutional principle; the **Write⇄Plan flip**
as a distinguished bar-end hinge on both surfaces, paired 1:1,
lazily born, never nagging; side-by-side as BM4's own ticket. Pairing
is schema (Nick's explicit go); the existing Plan machinery is
reconciled in BM1's brief before any build. FX10's named return ships
now unchanged.
